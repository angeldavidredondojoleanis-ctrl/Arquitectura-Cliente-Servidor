Arquitectura Cliente/Servidor

La arquitectura Cliente/Servidor divide las responsabilidades entre el cliente (interfaz de usuario) y el servidor (procesamiento y almacenamiento de datos). Este modelo permite escalabilidad, seguridad y modularidad en aplicaciones modernas.

Ejemplo: Tu navegador solicita una página web (cliente) y el servidor responde con HTML, CSS y JavaScript.

Diseño Centrado en el Usuario

El diseño centrado en el usuario busca crear interfaces intuitivas, accesibles y útiles. Se enfoca en la experiencia del usuario (UX) y en la facilidad de uso (usabilidad).

  • Interfaz clara y consistente
  • Accesibilidad para todos los usuarios
  • Validación de necesidades mediante pruebas

Importancia de la Investigación en el Entorno

La investigación en el entorno permite identificar requisitos reales y adaptar el sistema a las necesidades de los usuarios y del contexto. Esto asegura que el producto final sea relevante y funcional.

Ejemplo: En una terminal de transporte, investigar horarios, rutas y disponibilidad ayuda a definir los módulos del sistema.
